 1. Read-Only Shareable Links (The "Gallery" Feature)
  Verdict: High Priority / Best Practice
   * Why: Productivity tools thrive on "Proof of Work." If I build a beautiful mind-map of a project, I want to send a link to my boss or client to show
     them—without giving them the keys to edit it.
   * Monetization Idea:
       * Free: Can share 1 active space at a time.
       * Pro: Unlimited shareable links + Password protection for links.
   * The Cyberia Twist: Call them "Published Sectors." The link opens a web-view where the physics still works (you can play with the nodes), but you can't
     delete or add anything.


 2. Live Collaboration (The "Multiplayer" Mode)
  Verdict: The "Game Changer" / High Complexity
   * Why: This moves Cyberia from a "Personal Journal" to a "Team Whiteboard" (Miro/FigJam competitor). It's the #1 reason people pay for Pro software.
   * The Physics Challenge: Syncing physics in real-time is hard. If I pull a node left and you pull it right at the same time, the "spring forces" need to
     resolve smoothly across both screens.
   * Pro Feature Only: Definitely. This requires high-bandwidth server usage (WebSockets).
   * Brainstorm Idea: Imagine seeing your teammate's cursor as a small "spark" or "drone" flying around the workspace moving objects with you.

3. Version History (The "Time Machine")
  Verdict: Professional Standard
   * Why: If a user accidentally deletes a complex stack or mess up their layout, they need a way to go back.
   * Monetization Idea:
       * Free: 24-hour undo history.
       * Pro: 30-day "Snapshot" history.

"Show me moodboard ideas for..." $\rightarrow$ Route to search_pinterest or an image search.
